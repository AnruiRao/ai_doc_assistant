# 013: RRF 轻量融合

**日期：** 2026-06-30
**状态：** ✅ 已实施（默认关）

## 背景

`search_raw()` 在 Query Rewrite 开启时生成多条子查询，每条子查询返回一个独立排序列表。原先用 `seen` set 简单去重合并，丢失了各子查询的排名信息。

## 方案

实现 Reciprocal Rank Fusion（RRF）函数 `rrf_fuse()`，对多条子查询的排序结果进行倒数排名融合：

```
RRF(d) = Σ [ 1 / (k + rank(d)) ]
```

- k = 60（论文经典值）
- 按 `(source, chunk_index)` 去重
- RRF 不设独立开关，随 QR 自动联动

## 评估结果

| 实验 | Faithfulness | 结论 |
|---|---|---|
| Baseline | 0.6788 | — |
| Reranker only | **0.7883** | 最优配置（默认开） |
| QR+RRF+Reranker | 0.6692 | 单一检索策略无收益 |

## 根因分析

RRF 的典型应用场景是融合**不同检索策略**的结果（如向量搜索 + BM25 关键词搜索）。但在本项目当前架构下，QR 的 3 条子查询都搜索**同一个 Chroma 向量库**，返回的文档集高度重合，RRF 仅微调排序顺序但无法改变 Top-K 文档集合。

评测结果证实：RRF 在当前架构下无实际收益（甚至轻微拉低分数）。

## 结论

1. RRF 代码已实现并保留，供未来 Hybrid Search 使用
2. 默认关闭 QR（连带 RRF），当前最优配置为 Reranker 精排
3. 未来添加 BM25 等其他检索策略后，RRF 基础设施可直接启用
